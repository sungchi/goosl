# 구슬치기 WebGL 미니게임

외부 이미지나 3D 라이브러리 없이 WebGL 2 셰이더로 청록 유리, 내부 컬러 리본, 굴절, 하이라이트와 프리즘 그림자를 표현합니다. 기본 화면은 점수를 계속 누적할 수 있는 구슬치기이고, 자유 모드에서는 기존 샌드박스를 사용할 수 있습니다.

## 실행

프로젝트 루트에서 정적 서버를 실행한 뒤 브라우저로 접속합니다.

```bash
python3 -m http.server 4173
```

```text
http://localhost:4173/demo/
```

## 게임 규칙과 조작

- 구슬치기: 아래 발사 구슬을 쏠 방향으로 밀었다가 놓아 6번의 기회 안에 원 안의 7알을 밖으로 밀어냅니다. 미는 동안 파워가 표시되며, 7알을 모두 빼기 전에 발사 구슬이 원 안에서 멈추면 실패입니다. 살아남은 발사 구슬은 멈춘 그 자리에서 다음 기회를 이어갑니다.
- 7알을 모두 빼면 성공입니다. 발사 구슬도 원 밖에 살아 있으면 `이어서 계속하기`로 같은 위치에서 새 목표 7알과 기회 6번을 받아 점수를 14/7, 21/7처럼 계속 누적할 수 있습니다.
- 첫 시작과 이어하기에서 목표 구슬 7알이 차례로 떨어져 배치됩니다.
- 자유: 구슬을 직접 드래그해 던지고, 구슬 수·배경·일시정지·흩기를 조절합니다. 모바일에서는 기울기 센서 권한을 허용한 뒤 활성화한 순간의 자세를 기준으로 기기를 기울여 구슬을 굴릴 수 있습니다.
- 우측 상단 메뉴 버튼으로 모드·상태·조작 UI를 숨겨, 메뉴 뒤의 구슬도 직접 잡을 수 있습니다.
- Stacks와 같은 `ko/en` 번역 사전 및 브라우저 언어 감지 구조를 사용합니다. 첫 번째 브라우저 언어가 한국어 계열이면 한국어, 그 외 언어는 모두 영어가 기본이며 `?lang=ko`·`?lang=en`으로 검수 언어를 지정할 수 있습니다.
- 공유 썸네일도 언어에 맞춰 한국어 `thumbnail.png`와 영어 `thumbnail-en.png`를 사용합니다.
- 두 모드 모두 마찰로 에너지가 소진되면 속도를 정확히 0으로 고정합니다.
- 현재 누적 점수와 성공·실패 이유를 텍스트로 공유할 수 있습니다.

## GitHub Pages

비공개 저장소 `sungchi/marble`은 개발 소스만 보관하고 Pages를 활성화하지 않습니다. 공개 저장소 [`sungchi/goosl`](https://github.com/sungchi/goosl)이 Pages 전용 배포 대상입니다.

`.github/workflows/pages.yml`은 `main`이 갱신될 때 `demo`를 프로덕션 사이트로 빌드합니다. 빌드 과정에서 로컬 비교 버튼과 이전 버전 전환 코드를 제거하고 개선 버전(`visualVersion: 1`)만 남긴 산출물을 `goosl/main`에 강제 동기화합니다. 배포 인증에는 `goosl`에만 쓰기 가능한 deploy key와 `marble` Actions 비밀값 `GOOSL_DEPLOY_KEY`를 사용합니다.

`goosl` 저장소의 Pages Source는 **Deploy from a branch → main → /(root)**로 설정합니다. 사용자 사이트의 사용자 지정 도메인이 `plan9.kr`이면 최종 주소는 `https://plan9.kr/goosl/`이며, 프로젝트 사이트에는 별도의 `CNAME`을 두지 않습니다.

## 게임 적용을 고려한 구성

- 모든 구슬을 하나의 instanced draw call로 렌더링
- 구체 메시 대신 2-triangle billboard와 해석적 구면 셰이딩 사용
- 구 내부 광선과 베인의 교차점을 해석적으로 구해 유리 껍질과 분리된 3D 컬러 베인을 합성
- 이동 거리와 방향으로 3D quaternion 회전을 누적해 내부 베인이 구름 운동에 결합
- 공기방울·이물질도 베인과 같은 object-space에 보관해 굴러갈 때 동일한 quaternion으로 회전
- 구슬마다 폭·굽음·오프셋과 단일/분할/이중 베인 스타일을 독립적으로 생성
- 베인의 크기를 줄이고 테이퍼·끊김·비대칭 분기를 추가해 형태 반복 감소
- 구슬 회전에 붙는 1~3개의 스크래치, 미세 피트, 굴절 흔들림과 요철 노멀로 구슬별 반사 거칠기·하이라이트 편차 적용
- 각 구슬을 공통 청록 계열 안에서 녹색~푸른색으로 미세하게 달리 착색하고, 맑은 바닷물처럼 중앙 투과광과 광학 밀도를 독립적으로 변화
- 밝은 테두리의 내부 공기방울과 작은 광물성 이물질을 구슬별로 확률 생성
- 그림자에 긴 확산 암부, 구슬마다 위치가 다른 밝은 초점 카우스틱과 일부 구슬에서만 나타나는 RGB 프리즘 호를 합성
- 순백색과 얇고 넓은 청록 모눈종이 배경을 추가 텍스처 없이 셰이더에서 즉시 전환
- 배경은 화면 크기가 바뀔 때만 별도 텍스처에 구워 매 프레임에는 저렴한 복사와 그림자만 합성
- Web Audio로 공유 굴림 노이즈와 충격량·크기·좌우 위치가 반영된 짧은 유리 충돌음을 실시간 합성
- 충돌음 발생률 제한과 단일 굴림 채널을 사용해 72개에서도 오디오 소스 수를 일정하게 유지
- 외부 텍스처·오디오 요청 0개, 구슬별 색/3D 회전/크기는 작은 instance buffer로 전달
- device pixel ratio를 2로 제한해 고해상도 디스플레이의 fill-rate 제어
- 120Hz 고정 물리 스텝, 속도 수면 임계값, 무한 라운드 누적 점수
- 2D 원 충돌과 Pointer Events 기반 마우스·펜·터치 입력 통합
- 자유 모드에서만 6~72개 구슬을 실행 중 조절 가능

실제 게임에서는 `app.js`의 렌더러와 물리 상태를 엔진의 업데이트 루프에 분리해 가져갈 수 있습니다.
